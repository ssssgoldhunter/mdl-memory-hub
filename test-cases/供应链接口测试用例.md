## 1. scWithdraw — 余额提现

### 接口信息

| 项目 | 内容 |
|---|---|
| 完整URL | `POST http://fac-fcss-uat.mcdchina.net/gateway/catering/api/ums/catering/trans/scWithdraw` |
| Controller | `WIthDrawTransController` |
| 请求方法 | `POST` |
| Content-Type | `application/json` |
| 功能 | 对指定账户发起余额提现，金额 **元** 为单位（内部自动转分） |

### 请求参数

| 字段 | 类型 | 必填 | 长度 | 说明 |
|---|---|---|---|---|
| `userId` | String | 否 | | 用户编号，与 `cardCode` 二选一 |
| `cardCode` | String | 否 | | 卡号，与 `userId` 二选一 |
| `merchantId` | String | **是** | 15位 | 商户号 |
| `transNo` | String | **是** | ≤50 | 交易单号（调用方保证唯一） |
| `txnTime` | String | **是** | 14位 | 交易时间，格式 `yyyyMMddHHmmss` |
| `amount` | String | **是** | 1~22 | 交易金额（元），保留2位小数，≥0.01 |
| `fee` | String | 否 | 1~22 | 手续费（元），默认 `"0"` |
| `remark` | String | 否 | | 备注 |
| `bindId` | Long | 否 | | 提现账户绑定Id |
| `userBusinessType` | String | 否 | ≤50 | 用户业务类型 |
| `userBusinessCode` | String | 否 | ≤50 | 用户业务编码 |

### 测试用例

#### TC-1.1 正常提现 — 用 cardCode

```json
{
    "cardCode": "MCD99900001",
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_001",
    "txnTime": "20260622140000",
    "amount": "100.00",
    "remark": "测试提现-正常场景"
}
```

**预期结果**：
- `code = "0000"` — 提现受理成功
- `msg = "success"` 或类似
- 返回中包含 `consumeId`、`transNo`、`status` 等信息

#### TC-1.2 正常提现 — 用 userId

```json
{
    "userId": "MCD202600001",
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_002",
    "txnTime": "20260622140000",
    "amount": "50.00",
    "remark": "测试提现-userId"
}
```

#### TC-1.3 流水号重复（防重）

1. 先调用 TC-1.1 成功
2. 用完全相同参数再调一次

**预期结果**：
- `code ≠ "0000"`（例如 `"code": "01"`，`"msg"` 包含"流水号重复"）

#### TC-1.4 金额为0

```json
{
    "cardCode": "MCD99900001",
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_003",
    "txnTime": "20260622140000",
    "amount": "0.00"
}
```

**预期结果**：`code` 为错误码，提示金额必须大于0.01

#### TC-1.5 金额格式错误（小数位数>2）

```json
{
    "cardCode": "MCD99900001",
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_004",
    "txnTime": "20260622140000",
    "amount": "100.123"
}
```

**预期结果**：参数校验失败

#### TC-1.6 手续费传递

```json
{
    "cardCode": "MCD99900001",
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_005",
    "txnTime": "20260622140000",
    "amount": "100.00",
    "fee": "0.50",
    "remark": "测试提现-带手续费"
}
```

**预期结果**：`code = "0000"`，手续费会被计入处理

#### TC-1.7 userBusinessType + userBusinessCode

```json
{
    "cardCode": "MCD99900001",
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_006",
    "txnTime": "20260622140000",
    "amount": "200.00",
    "userBusinessType": "供应链提现",
    "userBusinessCode": "gyl_tx"
}
```

## 2. scWithdrawResultIm — 提现结果实时查询

### 接口信息

| 项目 | 内容 |
|---|---|
| 完整URL | `POST http://fac-fcss-uat.mcdchina.net/gateway/catering/api/ums/catering/query/scWithdrawResultIm` |
| Controller | `WithDrawQueryController` |
| 请求方法 | `POST` |
| Content-Type | `application/json` |
| 功能 | 根据 **交易单号** 实时查询提现结果（实时查上游） |

### 请求参数

| 字段 | 类型 | 必填 | 长度 | 说明 |
|---|---|---|---|---|
| `merchantId` | String | **是** | 15位 | 商户号 |
| `transNo` | String | **是** | ≤50 | 要查询的交易单号 |
| `receiveUserStoreCode` | String | 否 | | 用户自定义收款门店编号 |
| `userBusinessType` | String | 否 | | 用户业务类型 |
| `userBusinessCode` | String | 否 | | 用户业务编码 |

### 测试用例

#### TC-2.1 查询已成功的提现

```json
{
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_001"
}
```

**预期结果**：
- `code = "0000"`
- `status` 表示提现结果（S=成功, F=失败, P=处理中）
- 返回 `amount`、`balance`、`bankTransNo` 等信息

#### TC-2.2 查询不存在的 transNo

```json
{
    "merchantId": "320151MDC000001",
    "transNo": "TRANS_NOT_EXIST_001"
}
```

**预期结果**：`code` 为非0000，msg 提示未找到记录

#### TC-2.3 merchantId 与 transNo 不匹配

```json
{
    "merchantId": "320151MDC999999",
    "transNo": "TEST_WD_20260622_001"
}
```

**预期结果**：`code` 为非0000（找不到记录或无权限）

#### TC-2.4 查询处理中的提现

```json
{
    "merchantId": "320151MDC000001",
    "transNo": "TEST_WD_20260622_005"
}
```

**预期结果**：`status = "P"` 或未返回结果（取决于实时状态）

## 3. scQueryTransDetail — 提现记录明细查询

### 接口信息

| 项目 | 内容 |
|---|---|
| 完整URL | `POST http://fac-fcss-uat.mcdchina.net/gateway/catering/api/ums/catering/query/scQueryTransDetail` |
| Controller | `WithDrawQueryController` |
| 请求方法 | `POST` |
| Content-Type | `application/json` |
| 功能 | **分页** 查询指定 cardCode/userId 的提现记录明细。`transType` 固定为 `WD` |
| 是否分页 | **是** — `pageNum`/`pageSize`，最大 pageSize ≤ 100 |

### 请求参数

| 字段 | 类型 | 必填 | 长度 | 说明 |
|---|---|---|---|---|
| `merchantId` | String | **是** | 15 | 商户号 |
| `userId` | String | 否 | | 用户编号 |
| `cardCode` | String | 否 | | 卡号 |
| `startDate` | String | **是** | 14 | 开始时间 `yyyyMMddHHmmss` |
| `endDate` | String | **是** | 14 | 结束时间 `yyyyMMddHHmmss` |
| `pageSize` | String | **是** | 1~10 | 每页数量（纯数字） |
| `pageNum` | String | **是** | 1~4 | 第几页（纯数字） |
| `transType` | String | 否 | | **必须传 WD** |
| `rank` | String | 否 | | 排序 `1`=正序 `0`=倒序 |
| `queryMode` | String | 否 | | `0`=按明细 `1`=按订单 |
| `businessCode` | String | 否 | | 企业代码 |
| `receiveUserStoreCode` | String | 否 | | 自定义收款门店编号 |

### 测试用例

#### TC-3.1 正常查询 — 按时间段查所有记录

```json
{
    "merchantId": "320151MDC000001",
    "cardCode": "MCD99900001",
    "startDate": "20260601000000",
    "endDate": "20260623000000",
    "pageSize": "20",
    "pageNum": "1",
    "transType": "WD",
    "rank": "0"
}
```

**预期结果**：
- `code = "0000"`
- `bizList` 为提现明细数组
- `total` 总数
- `pageNum`/`pageSize` 符合传入

#### TC-3.2 按 userId 查询

```json
{
    "merchantId": "320151MDC000001",
    "userId": "MCD202600001",
    "startDate": "20260601000000",
    "endDate": "20260623000000",
    "pageSize": "10",
    "pageNum": "1",
    "transType": "WD"
}
```

#### TC-3.3 transType 不传/传错

```json
{
    "merchantId": "320151MDC000001",
    "cardCode": "MCD99900001",
    "startDate": "20260601000000",
    "endDate": "20260623000000",
    "pageSize": "10",
    "pageNum": "1",
    "transType": null
}
```

> **注意**：代码校验 `if (req.getTransType() == null || !CommonConstants.TRANS_TYPE_WD.equals(req.getTransType()))` 要求 transType **必须为 WD**

**预期结果**：`transType = null` 时，返回错误 `"交易类型异常"`

#### TC-3.4 pageSize 超过 100

```json
{
    "merchantId": "320151MDC000001",
    "cardCode": "MCD99900001",
    "startDate": "20260601000000",
    "endDate": "20260623000000",
    "pageSize": "200",
    "pageNum": "1",
    "transType": "WD"

```

**预期结果**：参数校验失败，返回错误（pageSize 不能 > 100）

#### TC-3.5 时间范围超过1年

## 附录：通用错误码对照

| 场景 | 预期 code | msg 说明 |
|---|---|---|
| 正常处理 | `"0000"` | 对应success/成功 |
| 参数错误 | `"02"` | 字段校验失败 |
| 流水号重复 | `"01"` 或 `"1292B00000xxx"` | 流水号已存在 |
| 卡号 userID 不匹配 | `"1292B00000xxx"` | 卡号与userId校验失败 |
| cardCode/userId/businessCode 全为空 | `"02"` | 不能同时为空 |
| 时间范围超过限制 | `"1292B00000xxx"` | 不能超过1年 |
| 时间格式错误 | `"1292B00000xxx"` | 要求 `yyyyMMddHHmmss` |

---

> **测试注意事项**
>
> 1. `scWithdraw` 每次调用 **transNo 必须唯一**，否则会返回流水号重复
> 2. 金额为 **元** 为单位，代码内部自动 `*100` 转换为 **分**
> 3. `scQueryTransDetail` 的 `transType` **必须传 `"WD"`**，否则会报交易类型异常
